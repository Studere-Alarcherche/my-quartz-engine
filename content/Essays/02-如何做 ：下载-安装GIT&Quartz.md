## 🛠️ 前置准备

在一切开始之前，你需要完成环境的“静默注入”。只需安装，无需打开：

- **Node.js**：前往 [nodejs.org]，下载并一直“下一步”无脑安装（选 **LTS** 版本） 最稳定。
    
- **Git**：前往 [git-scm.com]，下载安装。
    

---

## 🧱 动作 1：拉终端

在你的电脑上找一个干净的地方（比如 *_D:*_ 盘），新建一个文件夹，命名为 **MyGarden**（或者你喜欢的任何名字）。

进入这个文件夹，在顶部的地址栏里直接输入 **cmd**，然后按下回车。此时，你会看到一个黑色的窗口跳出来。这就是后台终端（即 **CLI**）。

## 📥 动作 2：拉取引擎源码 (Clone)

在这个黑窗口里，复制并粘贴这行代码，然后回车：

`git clone https://github.com/jackyzha0/quartz.git .`

> 💡 意思是“把 **Quartz** 的源码直接倾倒在这个文件夹里”，而不是再套娃建一个新文件夹。

## ⚙️ 动作 3：组装引擎零件 (Install)

源码拉下来后，只是一张图纸。我们需要 **Node.js** 去全网抓取真正的零件。继续输入并回车：

`npm install`

接下来，屏幕上会疯狂滚动各种绿色的进度条和英文字符。等待。系统正在为你下载底层编译文件（它们会被存放在一个叫 **node_modules** 的隐藏文件夹里）。等代码停止跳动，组装就完成了。

## 🧹 动作 4：初始化场地 (Create)

敲下这行初始化指令：

`npx quartz create`

系统会弹出几行英文问你想要什么模板。我们使用键盘的上下方向键，选择 **Empty (空模板)**，然后回车。

> 后面自己问豆包设计安排相关 **CSS TXV** 后端文件

## 🔥 动作 5：第一次本地点火 (Ignition)

为了确认这台物理引擎完好无损，敲下最后一行指令：

`npx quartz build --serve`

打开你的电脑浏览器，在网址栏输入 **localhost:8080** 并回车。**MyGarden** 文件夹，从此时应该至少有 **content**（用来放笔记的空壳）、**quartz.config.ts**（用来调配色的控制台）等文件。

## 🔌 补丁：Obsidian Git 神级插件

### 动作 1：插件安装

打开 **Obsidian** 的 **设置 (Settings)** -> 左侧找到 **第三方插件 (Community plugins)**。如果你是第一次用，点击关闭“安全模式 (**Safe mode**)”。点击 **浏览 (Browse)**，在搜索框输入 **Obsidian Git**。

### 🎛️ 动作 2：把这个屏蔽了

**极其重要的一步：防卡死。** 在 **Obsidian** 左侧找到 **.gitignore** 这个文件打开，确认里面有一行写着 **node_modules/**。这意思是让 **Git** 别去管那几万个代码文件，不然你的 **Obsidian** 会直接卡瘫痪。

### ⚙️ 动作 3：设定备份法则 (核心参数调优)

我们需要进行三项“注入灵魂”的微调：

- **一：优雅的时间戳 (Commit Message)**
    
    在 **Vault backup commit message** 这一栏，填入：`vault backup: {{date}} {{time}}`。以后你在 **GitHub** 上看到的将不再是乱码，而是像 **vault backup: 2026-03-01 23:10** 的时间戳。
    
- **二：自动与手动的哲学 (Auto Backup)**
    
    懒人模式（全自动）：在 **Vault backup interval (minutes)** 里填入 **30**。
    
    **推荐手动**：填入 **0**（关闭自动备份）。我强烈建议你保留这种**“物理推杆”的仪式感**。只有当你觉得“这段普鲁斯特的解析已经逻辑闭环”时，再由你亲手按下发送键。
    
- **三：静音模式 (Disable Notifications)**
    
    向下滚动，找到并开启 **Disable notifications**。
    

### ⚡ 动作 4：绑定“肌肉记忆” (终极发射快捷键)

我们不需要去点面板上的按钮，我们要把发射指令刻进肌肉记忆里。打开 **Obsidian** 设置 -> **快捷键 (Hotkeys)**。搜索 **Git: Commit all and push**。为它绑定一个你觉得最顺手的快捷键，比如 **Ctrl + Q**。

到这里差不多 也就结束了。最后，剩下的就是网站的渲染与打磨。你想让它变得有多好看，全凭你的直觉，但一定要记得**克制**。更重要的是，在不断的折腾中，你会逐渐养成一种直觉：解决一个问题需要花多久？加上 AI 辅助后它会有多复杂？在这个过程中你还会犯多少低级愚蠢的错误？这种对“复杂度”与“时间”的清晰认知，带来了一种实实在在的**实体感**。这种对自我的掌控与对时间的敬畏，或许才是瞎折腾折腾最大的好处之一。不管怎么说，这总比刷短视频要值得。

有时候，当你为了部署一个环境、折腾一项配置而大费周章，长舒一口气后，看着眼前的成果，脑海里难免会冒出一个反向的拷问：**浪费这么多时间做这些，到底有什么意义？**

在这个被技术焦虑裹挟的时代，我们常常会产生一种错觉：仿佛全球 80 亿人已经有 70 亿熟练掌握了 AI。这种随时会被超越的紧迫感是真实的，也必须要有。但当我们冷静下来，真正将 AI 深入融合到自我构建中的人，其实寥寥无几。

折腾这些的底线意义，或许只是学会了一点环境部署。但更重要的，是它逼迫我去思考：**有了 AI，我到底能干什么？** 我不需要成为代码生成器，但我必须清楚，这段代码是为了实现什么目的？是为了拉取哪种颜色，还是渲染特定框架？每一次报错，你得看懂哪里有问题。如果不具备这种“知其所以然”的纠错能力，那折腾技术就只是在浪费时间。

更残酷的真相是，纵使“赛博躯壳”做得再华丽，如果没有自己真正想写、想记、想完成的东西，空有框架也毫无意义。对我而言，这个项目很简单：**八面受敌，读好追忆。**

三四年前，我脑海中其实还没有这种宏大的构想。那时候更多的只是一种单纯的炫耀——四处宣告“我读完了《追忆》”。但在我真真切切活着的进程中，随着经历的人物、地点以及心灵感官的丰富，这个原本用来装点门面的谈资，突然在我体内膨胀成了一种贪婪的欲望。哪怕直到今天，我依然没学会法语，英语也不怎么样，但这不妨碍我想象那个能够自动跳转、具有生命感的《追忆》数字建筑。这确实是一种自说自话的狂妄，但有的“逼”装久了，也就弄假成真了。人总需要保留一点不切实际的念想，不然就难以感到感到幸福。

我依然会连续几个月陷入摆烂，不停地刷手机。但在虚度光阴之后，我内心最深处的痛感，一定是因为《追忆》的构建没有取得进展。这就是那把锚，它证明了这不仅是兴趣，而是执念。

为了这个执念，我需要实现跨媒介的跳转：从一张图片链接到艺术史、画家、技法，最后与音乐旋律对比。现实的引力依然沉重，比如图床的负担、音乐挂载的成本，但我并不着急。AI 与技术的发展正在加速，瓶颈早晚会被解决。也不是实在瓶颈，是对我而言，不值得浪费太多时间的瓶颈。相较于这个而言，现在的我更重要的仍然是语言和文本  技术和表达是这个过程当中更微小的一部分，虽然也重要

在这个过程中，唯一无法被 AI 取代的是**判断力与审美抉择。** 你必须亲自下场，去感知哪些节点之间具有隐秘的相似性。当你捕捉到那个令你灵魂震颤的连接时，那种满足感足够抵消所有疲惫。
